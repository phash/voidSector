## Issue #106: Main-Lower Redesign

The `main-lower` area (currently: SectorInfo + StatusBar + NavControls) becomes a 3-zone layout.

### Layout

```
┌──────────────────────────────────────────────────┐
│  SECTOR: (x,y) | TYPE | PILOTS | ORIGIN | AP | FUEL | CR  │  ← Compact info bar
├──────────────────────────────────────────────────┤
│              ↑  ←  ↓  →   [SCANS] [BUILD]       │  ← Nav + Action Buttons
│         [JUMP TARGET] [AUTOPILOT]                │  ← Jump options
├────────────────────┬─────────────────────────────┤
│  SHIP SYSTEMS      │  COMBAT SYSTEMS             │
│  ─────────────     │  ─────────────              │
│  DRV: Mk2  ✓      │  WPN: Laser Mk1    DMG: 12  │
│  SCN: Mk1  ✓      │  SHD: ████░░░░ 45/80        │
│  CRG: 24/50       │  DEF: Point Defense          │
│  ARM: HP 60/60    │  ECM: ---                    │
│  ENG: SPD 2       │                              │
└────────────────────┴─────────────────────────────┘
```

### Zones

1. **Top bar** — Merged SectorInfo + StatusBar in a single compact row (position, sector type, AP bar, fuel bar, credits)
2. **Center** — NavControls (direction buttons, scan buttons, build buttons, jump/autopilot options)
3. **Bottom-left** — `ShipStatusPanel` (new component): drive, scanner, cargo capacity, armor/HP, engine speed, fuel. Data from `ship.stats` and `ship.modules`.
4. **Bottom-right** — `CombatStatusPanel` (new component): weapon type + damage, shield status with schematic bar, point defense, ECM. Data from `ship.modules` filtered by weapon/shield/defense categories.

### SHIP-SYS Changes

SHIP-SYS monitor becomes settings-only:
- Color profile selector (stays)
- Future: keybindings, audio, display settings
- The SchematicView (ship diagram with slots) moves conceptually into ShipStatusPanel/CombatStatusPanel
- Module management and Hangar remain as SHIP-SYS modes (modules, hangar)

## Issue #103: Tech-View Main+Detail Split

When TECH is active in main screen, it uses the existing `main-upper` split layout (same as NAV split with radar+detail).

### Main-Grid (left panel)

- Tech tree organized by category (ANTRIEB, SCANNER, WAFFEN, etc.)
- Compact module list per category with status icons:
  - Green checkmark = unlocked/free
  - Blue "BP" = blueprint available
  - Yellow spinner = researching
  - Gray lock = locked
- Click on any module to select it (highlights row)
- Active research shown at top with countdown

### Main-Detail (right panel)

- Selected module detail view:
  - Module name, tier, category
  - Primary + secondary effects with values
  - Research cost (credits, ore, gas, crystal, artefact)
  - Research duration
  - Prerequisites (which modules must be researched first)
- Action buttons:
  - `[FORSCHUNG STARTEN]` — if at home base, has resources, no active research
  - `[ABBRECHEN]` — if this module is being researched
  - `[ABSCHLIESSEN]` — if research complete
- If no module selected: placeholder text "Select a module to view details"

### Main-Lower

Standard main-lower with ship status (same as NAV view).

## Issue #104: Base-View Main+Detail Split

When BASE-LINK is active in main screen, same split layout.

### Main-Grid (left panel)

- Base overview showing all structures as a grid/list:
  - Each structure: icon/label, tier, status (ACTIVE/IDLE/UPGRADING)
  - Structures: Kommando-Kern, Comm Relay, Mining Station, Lager, Handelsplatz, Fabrik, Kontor, Forschungslabor
- Base name at top with rename option
- Credits display
- "No base" state if no base built

### Main-Detail (right panel)

- Selected structure detail:
  - **Storage:** capacity, contents, transfer buttons, upgrade option
  - **Factory:** active recipe, progress bar, collect button, recipe selector
  - **Kontor:** order form, active orders list
  - **Trading Post:** upgrade option
  - **Other structures:** basic info, tier, status
- If no structure selected: general base summary (total structures, coordinates)

### Main-Lower

Sector info of the home base sector.
